# ADR: バトル画面ではアイドル名・グループ名・投票履歴を表示しない

- 日付: 2026-05-22
- ステータス: 採用

## コンテキスト

`/battle` 画面は「2人のアイドル写真を見て、好きな方をタップして投票する」UI。
従来は写真の上に以下を重ねて表示していた:

- 各パネルにアイドル名・グループ名のオーバーレイ (`BattlePanel` 内のネームプレート)
- 画面下に過去3件の投票結果を出す `VoteHistoryPanel`

しかし「写真が見えなくなる」という UX 上の問題が報告された。
顔・スタイル・雰囲気で判断するアプリにおいて、写真が覆い隠されるのは本末転倒。

## 決定

バトル中の画面からは以下を **すべて削除** する:

- パネル上のアイドル名・グループ名オーバーレイ
- `VoteHistoryPanel` (画面下の過去の投票結果)

代わりに:

- パネル `<button>` には `aria-label="<グループ> <名前> に投票"` を付与し、スクリーンリーダーと自動テストから識別可能にする
- 投票履歴は引き続き localStorage に保存する。閲覧は `/history` で行う
- 進捗インジケータ (`ROUND XX/10`) は写真を覆わない小さなチップなので維持する

## 影響

- `apps/web/src/routes/battle.tsx`: `VoteHistoryPanel` とネームプレートを削除
- `apps/web/src/routes/-__tests__/battle.test.tsx`: 名前表示・履歴表示の存在を確認していたアサーションを反転 (表示されないことを確認) し、ボタン特定は `aria-label` 経由に変更
- `docs/REQUIREMENT.md`: 「名前表示」を必須から外し、本 ADR にリンク

## 代替案

- **写真の隅に小さく表示する**: それでも写真の一部を覆う。判別は写真自体で行えるので不要と判断
- **タップ後の遷移中だけ名前を一瞬出す**: 次ペアの即時表示 (REQUIREMENT の「0.1 秒レベルで次表示」) と衝突するため不採用
- **ホバー/長押しで表示**: スマホファースト UI でホバーは使えず、長押しは投票という主動作の邪魔になるため不採用

名前を確認したい場合は投票後に `/history` で参照できるため、UX 上の損失は小さい。
